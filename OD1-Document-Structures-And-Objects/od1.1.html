Implement and Manipulate Document Structures and Objects

Create the document structure by using HTML

January 14, 2019

In this article, we'll be covering the first skill covered under the first objective domain Implement and Manipulate Document Structures and Objects: Create the document structure by using HTML.

According to Microsoft, the skills measured are as follows:

Structure the UI by using semantic markup, including markup for search engines and screen readers, such as Section, Article, Nav, Header, Footer, and Aside; create a layout container in HTML

This page has been built using all of the specifically mentioned elements, as well as some other new elements introduced in HTML 5. Information regarding each elements will be included in it's own section, and I'll try to do my best to explain when and why to use each element.

An important thing to keep in mind is that, though I have styled these pages, these new elements are meant to convey the structure of the document and not the appearance of the content.

The Section Element

The <section> element, according to W3, represents "a generic section of a document or application. A section, in this context, is a thematic grouping of content, typically with a heading."

Common comparisons include chapters in a book or a home page being split into an introduction, news, and contact information.

The first question I asked myself was "How is this different than a 'div' tag?" After all, "a generic section of a documnet" sounds like a div to me. The major difference is that the section element is meant to group content that are contextually related.

As an example, each element on this place is logically grouped together into it's own section. Everything in this "Section Element" section, and all of it's contents, belong to one section element. The "Article" section belongs to anotther

So, when do you use a div or span? Basically, when you're wanting to call out a specific piece or group of content for styling purposes only. All of the italicized text was styled via a class on a <span> tag.

The Article Element

From the W3C Specification, the article element is defined as follow:

The article element represents a self-contained composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.

A bit of a mouthful, I believe the short version is that an article is any part of your document that is still coherent when everything else is taken away from your page.

The classic comparison would be to a new papers column, or an article from a magazine. You can clip out a particular story (or comic) from the news paper and the same information is still conveyed.

Articles and Sections

So, here's an interesting situation. An <article> tag can contain any number of <section> tags. Additionally, <section> tags can contain any number of <article> tags. And both elements can contain other instances of the same element. Having sections within sections makes lexical sense. Articles within articles less so, at least for me...but you can certainly do it.

So, should this page be a "Skills Measured" section with an article for each skill, or is it a single article with a section for each skill?

I focused on my intent that the contents on this page were meant to be kept together as one big article. While some sections could certainly stand on their own merit, others (such as this section) cannot.

I think the specifications for these elements, and that fact that each can contain the other, suggests that there's more than one "right" way...which makes perfect sense. Sections of news papers consist of articles, and each article in a newspaper can contain multiple sections (like an introduction, body, and summary).

The Nav Element

Again, the description from W3

The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links.

Simple enough, right? This element is just used to denote a group of links used for navigational purposes.

There are, however, situations where you should not use the <nav> element to group a list of links. If you have a list of links, such as references or "More like this" links, those should not be grouped by the nav element.

Any "global" navigation links, such as those at the top of this page, or links used to help navigate the structure of the current document like the ones on the left column, should be contained within the nav element.
